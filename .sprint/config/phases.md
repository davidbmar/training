## Sprint Cycle (every sprint)

Each sprint follows this cycle:
1. **Build** — Agents implement features from the sprint brief
2. **Review** — As PM and customer, review all delivered components
3. **Customer Testing** — Use Playwright to simulate real user journeys, find UX gaps
4. **Backlog Triage** — Bugs and feature requests from review go into backlog
5. **Plan Next** — Use roadmap goals + prioritized backlog to write next sprint brief

## Roadmap Extension

On Sprint 5 (or when 1 planned sprint remains), extend the roadmap with 4+ additional sprints based on progress and learnings.

## Phase Groupings (customize per project)

### Phase 1: Foundation (Sprints 1-2)
Core functionality, basic architecture, initial tests.

### Phase 2: Features (Sprints 3-4)
Key features, API, integrations, error handling.

### Phase 3: Polish (Sprints 5-6)
UI/UX, testing, deployment, public release.
